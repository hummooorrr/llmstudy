# Pro-RAG 用户使用手册

欢迎使用 Pro-RAG！这是一款基于你的私有资料库进行智能问答和文档起草的助手，帮助你**把散落在各处的资料变成可以直接回答问题、直接生成公文的知识**。

本手册面向直接使用网页的使用者，不涉及技术细节。请按章节阅读即可上手。

---

## 一、它能帮你做什么

一句话：**把资料"喂"给它 → 它替你检索、回答、写稿**。

具体来说：

1. **建立自己的知识库**：把合同、通知、报告、手册、图片、扫描件等丢进去，它会自动读懂并记住。
2. **基于知识库问答**：问问题，它会从你的资料里找答案，并附上"引用来源"，让你一眼看到答案出自哪份文件、哪一页。
3. **快速定位文件**：输入一个关键词或一个问题，它告诉你"这件事最有可能出现在这几份文件里，具体是哪几段"。
4. **一键起草文档**：给它一句指令（例如"根据新规定写一份风控整改方案"），它会结合你的资料自动生成 Markdown 或 Word 文档，支持多轮修改。
5. **上传即生成**：遇到"新通知刚下发、需要立刻出一版对应稿件"的情况，上传 + 指令 → 直接出稿。

## 二、进入页面时先了解这些

打开页面后，你会看到：

- **左上角 Logo「Pro-RAG」**：这是助手的名字。
- **顶部「会话」标签**：显示"对话 xxxxxxx / 生成 yyyyyyyy"两个会话编号。
  - **对话** 是你当前在「RAG 对话」里使用的会话；
  - **生成** 是你当前在「文档生成」里使用的会话。
  - 两者**互相独立**，互不干扰——在对话里说的话不会跑进生成记忆，反之亦然。
  - 会话编号会被**自动保存在浏览器里**：刷新、关机重开，下次进来仍是同一个会话，之前的聊天记录和历史版本都还在。
  - 想换一段新对话/新起一篇稿件？分别点各自的「**+ 新会话**」按钮即可。
- **顶部 Tab 行右侧的「当前领域」工具条**：会跟着滚动一直"钉"在顶部，随时可切换。提供四个领域：
  - **银行风控**：默认，适合风控合规类文件与问答。
  - **历史**：适合历史类资料的问答和写作。
  - **教育**：适合教学设计、教研报告、课程方案、学生培养计划等教育类资料。
  - **医疗**：适合诊疗规范、病例分析、科室制度、健康教育材料等医疗类资料（涉及用药、剂量、诊断等敏感内容时以上传的资料为准，不要凭此编造临床建议）。
  - 切换后，助手会换用对应领域的语气和行文规范。**提问前请先切到合适的领域。**
  - 领域切换**只影响「RAG 对话」和「文档生成」两个标签页的回答/生成语气**；「知识库上传」和「文档定位」对领域不敏感，所有领域共用同一个知识库，不管当前挂在哪个领域都能检索到所有已入库的资料。
  - **不建议在同一个会话中途切换领域**：切换后只会改变下一条消息使用的提示词模板，**旧会话里的历史消息仍会被带进上下文**，容易出现新旧风格打架（尤其是「文档生成」的多轮改稿，会和"最小化增量修改"规则冲突，导致结构错位）。要换领域，请先点「**+ 新会话**」开一份新的再切。
- **四个主要标签页**（点击切换）：
  1. 知识库上传
  2. RAG 对话
  3. 文档定位
  4. 文档生成

下面逐个讲解。

---

## 三、【知识库上传】——先让助手"读"你的资料

点击顶部的「**知识库上传**」标签。

### 1. 上传文件到知识库

**操作步骤**：

1. 把文件**拖到虚线框里**，或者**点击虚线框**从本地选择文件，支持一次选多份。
2. 支持的格式：**PDF、Word（.doc / .docx）、Markdown（.md）、TXT、HTML、JSON、图片（jpg / png / bmp / tiff / gif）**。
3. 检查一下下方的文件列表，点「**上传入库**」即可。
4. 上传过程中会显示进度条，完成后会显示每个文件切成了多少"片段"。

**温馨提示**：

- 上传的原文件**默认保留 7 天**，7 天后会被系统自动清理。清理后这些原文件就无法"补录"了，但之前已经入库的知识仍然可以被问答和生成使用。
- 扫描件、带图片的 PDF 也可以直接上传，助手会自动识别其中的图片和表格。
- 文件比较大（尤其是含大量图片的 PDF）时，上传和入库可能需要等一会儿，请耐心等待"入库完成"的提示。

### 2. 存储清理（谨慎操作）

这张卡片用于手工清理历史文件：

- **清理过期上传文件**：删除上传目录中 7 天以上的原文件（不影响已入库的知识）。
- **清理过期生成产物**：删除你之前"生成文件"产生的 Markdown / Word 成品（7 天以上）。
- **清理全部过期文件**：以上两种一起来。

下方还有一个**"彻底清理该上传文件"**：

- 在输入框里填入**存储文件名**（不是你本机的原始名，是上传后系统生成的名，可以在"生成文件"卡片的下拉里看到），点红色按钮。
- **这会把这份文件从知识库里彻底抹掉**：本地原文件、向量库、关键词库全部删除，助手以后再也不会引用到这份文件。
- 仅在确认不再需要这份资料时使用，**无法撤销**。

## 四、【RAG 对话】——像聊天一样问问题

点击顶部的「**RAG 对话**」标签。

### 使用方法

1. 在底部输入框里输入你的问题，例如"新规对风控有什么影响？"。
2. 点「**发送**」或按回车。
3. 助手会**流式**（像打字一样逐字）输出答案。
4. 回答里会出现类似 `[^c1]`、`[^c2]` 的**角标**——这是**引用标注**。
5. 右侧「**参考文档**」面板会列出本次回答用到的所有引用，按 `c1、c2…` 编号，和答案中的角标一一对应。
6. **点击右侧任意一条引用卡片**，会弹出一个窗口，展示这个片段的**原文**、所在文件、页码，还包括它前后的相邻段落，方便你核对上下文。
7. 如果这条引用带有图片或表格截图，弹窗里也会显示图片。

### 多轮对话 & 历史会话

- **有记忆**：在同一个会话里继续提问，助手会记得之前说过什么，可以直接追问、让它改写、让它更精简（默认保留最近 30 条消息作为上下文，历史全量都会存在服务器上）。
- **持久化**：会话**不会因为刷新、关闭浏览器或服务器重启而丢失**。下次打开页面会自动回到你的最后一个会话。
- **历史会话侧栏**：卡片顶部有一条"**历史会话**"条，展示最近用过的所有会话。
  - 鼠标悬停能看到完整标题和更新时间。
  - **点击任意一条**，就能切回那个会话，之前聊过的消息会回填到聊天区，继续追问即可。
  - 每条会话后面有一个小 × ，**点一下就能删除**（会连带消息一起删，删完会自动开一个空白新会话）。
  - 助手会用你**第一条提问**的前几十个字自动生成会话标题，方便区分。
- **新开一个话题**：点输入框旁的「**新对话**」或侧栏的「**+ 新会话**」，会生成一个新的会话编号，助手从零开始，同时旧会话依然保留在侧栏里。

### 小技巧

- **问题越具体，答案越精准**。例如"新规对风控的影响"不如"2025 年反洗钱新规对小微企业贷款风控的影响"。
- 如果回答中没有角标，说明助手**没有在你的资料里找到明确依据**，这时答案可能不够可靠，请谨慎采信。
- 切换"银行风控 / 历史 / 教育 / 医疗"领域会改变助手的语气和格式，根据你问的内容挑合适的领域。

## 五、【文档定位】——先找文件再翻原文

点击顶部的「**文档定位**」标签。

这个功能不让助手"回答你"，而是让它告诉你：**这事儿在哪几份文件里写过？**

### 使用方法

1. 输入框里填入关键词或简短问题，例如"贷款展期"、"风险加权资产计算方法"。
2. 点「**搜索**」。
3. 结果会按相关度从高到低列出若干份文件，每份文件下面会展示：
   - 文件名和路径
   - 命中了多少个片段
   - 具体的高亮片段（让你确认就是你要的内容）
4. 这一步不会生成任何新的内容，只是帮你**快速锁定源文件**。

**适用场景**：

- 知道某个事情"肯定有文件写过但记不清在哪"。
- 想找到原件再自己阅读全文。
- 给「文档生成」环节**挑出指定的通知文件**（它会告诉你这个文件的存储文件名，下一步直接用）。

## 六、【文档生成】——让助手替你写稿

点击顶部的「**文档生成**」标签。这一页分成 **五张卡片**：一张会话管理卡片（最上方）、三张产出卡片、一张"多轮修改 & 版本历史"卡片（最下方）。

### 卡片 0：生成会话（最上方）

- 展示所有"生成"作用域下的历史会话，点击可切回任意一份旧稿件继续改。
- 切换到某个旧会话后，下方的"版本历史"会**自动回放**那份稿件的所有历史版本。
- 新开一份稿件：点「**+ 新会话**」。这份会话会独立保留，不影响原来的稿子。
- 想删除某份旧稿会话？点击它右上角的 ×（会同时删除历史消息，但已下载的 Word / Markdown 成品文件不受影响）。

### 卡片 1：流式预览（不保存）

最快的试写方式，**用于看效果，不会落盘**。

1. 输入一条自然语言指令，例如"根据新规定撰写一份风控报告"。
2. 点「**生成**」。
3. 助手会流式输出一整份文档，基于你的知识库里的相关材料。
4. 不满意？直接修改指令再生成一次。

> 注意：这里的输出**不会自动保存成文件**。如果要下载成品，请用下面两张卡片。

### 卡片 2：生成文件产物（Markdown / Word）

保留预览能力，同时把成品保存下来。

**必填**：

- 生成指令（输入框）。

**选填**（下方 4 个字段）：

| 字段 | 作用 |
|---|---|
| **指定通知文件名** | 强制让助手在起草时**优先引用这一份文件**。格式是"存储文件名"（不是原名），输入框支持下拉选历史上传过的文件。 |
| **输出格式** | Markdown（默认）或 Word（.docx）。 |
| **文档标题** | 成品文档的标题。留空会自动用"通知名 + 生成稿"或时间戳。 |
| **输出文件名** | 下载时的文件名。留空按标题自动生成。 |

**操作流程**：

1. 填好指令和可选参数。
2. 点「**生成文件**」，等待完成。
3. 下方会出现：
   - **文件预览**（片段）
   - **下载按钮**：点击即可下载 Markdown 或 Word。
   - **本次引用的参考文档**列表。
4. 想要微调？**直接滚动到最下面的"多轮修改 & 版本历史"卡片**，在"继续修改"输入框里写出修改意见（例如"把第三部分精简为 5 条要点"），助手会在**上一版基础上做最小化增量修改**，不会整篇重写。

> 生成产物默认保留 7 天，请及时下载。超期后无法再下载，但可以用同样的指令重新生成。

### 卡片 3：上传新通知并立即生成

**适用场景**：刚收到一份新通知/新规定，还没入库，就想马上基于它起草对应文稿。

1. 点击文件选择，挑一份本地新通知文件（格式同上传支持列表）。
2. 填写生成指令，例如"根据这份新通知起草一版合规整改方案"。
3. 选输出格式（Markdown / Word），如需要填文档标题和输出文件名。
4. 点「**上传并生成**」。
5. 助手会：
   - 自动把这份新通知**入库**（下次可直接在"参考文档"或"指定通知文件名"里使用）；
   - **自动把它当成指令文件优先引用**，结合知识库里的其他历史材料，直接产出一份成稿。
6. 完成后，同样提供下载按钮和引用列表。
7. 和"卡片 2"一样，后续修改请到下面的"多轮修改 & 版本历史"卡片继续迭代，**不必重复上传文件**。

### 卡片 4：当前会话 · 多轮修改与版本历史（重点）

这是对写长稿体验最大的一张卡片。

- **继续修改输入框**：写一句修改意见（例如"第二节多加两条风险点"、"把结尾改得正式一点"），选输出格式后点「**继续修改**」，助手会读取本会话下上一版的完整文稿作为上下文，然后**仅调整你明确提到的部分**，未提及部分原样保留，不整篇重写。
- **版本历史列表**：每次成功生成/修改都会在这里加一条 `v1 / v2 / v3…`，**按时间倒序**。每条包含：
  - 版本号（彩色标签）
  - 产出文件名 / 指令摘要
  - 生成时间
  - **下载按钮**：直接下载那一版的 Markdown 或 Word 文件（即便当前已经改到更新的版本，旧版文件仍在服务端，可随时回取）。
- **切回旧稿**：顶部的"生成会话"列表里点旧会话，版本历史会回放；下载任意一版直接替换当前稿件。
- **重新开始**：觉得一份稿件改得太乱了？点"生成会话"区的「**+ 新会话**」开一份全新的，旧稿件仍保留，不会被覆盖。

> 使用建议：大的结构性调整（改主框架、换行文风格）建议开新会话；局部微调（加段、删段、改某一章）就用"继续修改"。这样版本历史会更清晰。

## 七、关于引用和答案来源

这是 Pro-RAG 最重要的特性之一——**所有回答/生成的内容都尽量给出引用**。

- 在**对话**里：答案中的 `[^c1]` 和右侧引用列表一一对应，点击即可看原文片段。
- 在**生成文件**里：成品文档的末尾或侧边栏会展示本次用到的参考材料清单，也会有"引用编号"可以追溯到原文件。
- 如果回答里**没有角标**，或材料列表为空，说明助手**没有在你的资料里找到对应依据**——此时请自行判断答案是否可靠，不要盲信。

## 八、常见问题（FAQ）

**Q1：我上传了文件，问答时却说"没有找到相关信息"？**
A：先确认上传结果提示"成功"。如果依然查不到：
- 试试"文档定位"功能用关键词搜一下，看看文件是否真的在库里。
- 问题表述再具体一些，或换几个同义词。
- 如果是图片或扫描件，文字内容需要依赖 AI 识别，有时小字、低清会漏识，可以改用更清晰的文件。

**Q2：下载的文件打不开 / 找不到了？**
A：
- 生成产物保留 7 天，超期会被清理，重新生成即可。
- 下载按钮点开以后是 Markdown 或 Word 文件，用相应软件（VSCode / 记事本 / Word / WPS）打开。

**Q3：我想只让助手引用某一份文件，可以吗？**
A：可以。在"生成文件产物"或"上传并生成"卡片里，填入那份文件的**存储文件名**（输入框有下拉联想）。助手会**优先把这份文件的内容纳入参考**。

**Q4：为什么我换了个问法，答案不一样？**
A：这是正常现象。助手每次都会根据问题检索最相关的片段，问法不同命中的片段就不同。想更稳定的结果，建议在同一个会话下追问"按第一版继续补充…"。

**Q5：关了浏览器、重启电脑之后，之前的对话还在吗？**
A：**在**。对话和生成两种记忆都存在服务端数据库，关掉浏览器只会让页面"失去焦点"，不会删除任何记录：
- 再打开时，页面会默认帮你恢复到上一个会话；
- 想看更早的会话，点卡片顶部的**历史会话**条，列出全部历史会话，点击即可切回；
- 即使运维人员重启了服务、升级了应用，对话记忆也都在 MySQL 里不会丢。
- 只有当你主动点了会话上的 × 删除，或管理员手工清理数据库时，记录才会消失。

**Q6：误上传了敏感文件怎么办？**
A：进入「知识库上传」底部的「存储清理」卡片，输入该文件的存储文件名，点「彻底清理该上传文件」——原文件、向量库、索引都会立刻删除，助手之后不会再引用到这份文件。

**Q7：助手在回答里"编造"了不存在的信息怎么办？**
A：
- 先看回答里有没有角标 `[^cN]`。没有角标的段落表示没有依据，谨慎采信。
- 点引用卡片核对原文，如果原文里根本没有该说法，说明是模型的自发编写，请以原文为准。
- 把对应资料补充进知识库后再问，通常会得到更可靠的回答。

**Q8：同一个会话聊到一半切换领域，会发生什么？**
A：
- 系统**只会对你切换后新发的消息使用新领域的提示词**，旧的历史消息不会被清空，仍然会作为上下文参与本次回答。
- 这会带来两类风险：
  - **对话**里短追问可能被旧上下文"带偏"，新领域的身份没完全生效；
  - **文档生成**里用"继续修改"尤其容易翻车——新领域的结构模板会与上一版稿件的结构冲突，还会和"最小化增量修改"规则打架，可能出现结构错乱或默认文件名突然变掉。
- 推荐做法：要换领域，就点「**+ 新会话**」开一份新的再切；原来的会话和历史版本仍然保留在侧栏里，不会丢。
- 同一话题确实想听"换个视角的解读"（比如把一批资料从风控视角换到教育视角），可以直接在追问里写明"请以教育角度重新梳理"这种明确指令，比光切按钮更可靠。

## 九、推荐使用流程

对于**日常问答**：

1. 进入「知识库上传」上传相关资料 → 等待"入库完成"。
2. 切到对应领域（银行风控 / 历史 / 教育 / 医疗）。
3. 进入「RAG 对话」提问，核对右侧引用。
4. 不够精准就追问，必要时换问法。

对于**起草公文 / 报告**：

1. 先确保参考资料都已上传入库。
2. 如果有"指令文件"（例如上级新通知）：
   - 已入库 → 进入「文档生成 · 生成文件产物」，填写"指定通知文件名"。
   - 未入库 → 进入「文档生成 · 上传并立即生成」，直接上传并填指令。
3. 选好输出格式（Markdown 或 Word），点生成，得到 **v1**。
4. 滚动到最下方的「**多轮修改 & 版本历史**」卡片：
   - 在"继续修改"输入框里写修改意见（例如"把第二节合并进第一节"、"把结尾补一段展望"）。
   - 点「继续修改」→ 得到 v2、v3……每一版都能单独下载。
5. 满意后，在版本历史里点对应版本的「下载」即可。
6. 想另起一份？点顶部「+ 新会话」，旧稿件和所有历史版本都会保留在会话列表里。

对于**快速查资料**：

1. 进入「文档定位」。
2. 输入关键词 → 查看命中文件及片段。
3. 若需要进一步问答或生成，把文件名复制到相应功能里使用。

## 十、注意事项

- **文件大小**：单个文件最大 1024MB，请合理拆分。
- **保留策略**：原文件与生成产物默认保留 7 天，重要内容请及时下载或转存。对话/生成的消息**没有自动过期策略**，长期保留，以便随时回溯。
- **数据来源**：助手的回答只来自你上传的知识库 + 对话上下文，没有的信息它会说"没有找到相关信息"。
- **引用核对**：涉及合规、法律、金额等关键内容时，**务必点开引用卡片核对原文**再采用。
- **领域切换**：不同领域会影响行文风格和提示词，提问或生成前请先确认领域是否切对；领域只在「RAG 对话」和「文档生成」里生效，对知识库入库和文档定位没有影响。
- **浏览器缓存**：会话标识（chatId）存在浏览器 localStorage。如果使用隐私/无痕模式、清理浏览器数据、或者换了一台电脑打开，会被当成新用户，默认开一个新会话；但之前所有会话仍然在服务端，可以通过"历史会话"侧栏找回。
